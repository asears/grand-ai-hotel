## LLM Detection Markers (AVOID)

These words and patterns are commonly generated by LLMs and should be avoided in professional documentation to maintain authentic, human-written quality.

### Overused Adjectives

| ‚ùå Avoid | ‚úÖ Use Instead | Context |
|----------|----------------|---------|
| comprehensive | complete, full, detailed, thorough | "Complete test suite" not "Comprehensive test suite" |
| robust | reliable, stable, tested, proven | "Reliable error handling" not "Robust error handling" |
| seamless | smooth, integrated, unified | "Smooth integration" not "Seamless integration" |
| cutting-edge | modern, latest, current | "Modern architecture" not "Cutting-edge architecture" |
| leverage | use, utilize, employ | "Use this API" not "Leverage this API" |
| holistic | complete, integrated, unified | "Complete approach" not "Holistic approach" |
| innovative | new, novel, creative | "New solution" not "Innovative solution" |

### Formatting Patterns to Minimize

| Pattern | Context | When Appropriate |
|---------|---------|-------------------|
| **Bold emphasis** | Use sparingly | Tutorials, changelogs, announcements |
| üéØ Emojis | Use sparingly | Tutorials, READMEs, casual docs |
| Em-dashes ‚Äî like this | Avoid in code/standards | Acceptable in narratives |
| Excessive markdown | Avoid cruft | Keep professional docs clean |

**Rule:** In best practices documents, code files, and technical standards, prefer plain formatting. Save visual flair for tutorials, guides, and user-facing content.

## Semantic Versioning

Project follows [Semantic Versioning 2.0.0](https://semver.org/):

- **MAJOR.MINOR.PATCH** (e.g., `1.2.3`)
- **MAJOR**: Breaking changes
- **MINOR**: New features (backward compatible)
- **PATCH**: Bug fixes (backward compatible)

**Example:**
- `1.0.0` ‚Üí `1.1.0` (added feature)
- `1.1.0` ‚Üí `1.1.1` (bug fix)
- `1.1.1` ‚Üí `2.0.0` (breaking change)

## Code Comment Patterns

### TODO Comments

Format: `TODO(INITIALS): Description`

**Examples:**
```python
# TODO(AS): Refactor this method for better performance
# TODO(DOCS): Add example usage
# FIXME(AS): Handle edge case when input is None
# HACK(AS): Temporary workaround until API v2 is released
```

### Placeholder Comments

For removed dependencies or incomplete features:

```go
// TODO(DOCS): This is a placeholder implementation.
// For production use, see REMOVED_DEPENDENCIES.md for alternatives:
//   1. Custom Office Open XML parser (free, open source)
//   2. Microsoft Graph API (cloud-based, official)
//   3. LibreOffice headless (server-side conversion)
```

| Prefix | Meaning | Example |
|--------|---------|---------|

| `chore:` | Maintenance | `chore: update dependencies` |
| `security:` | Security fixes | `security: patch CVE-2024-1234` |
